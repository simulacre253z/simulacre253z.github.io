● Request & Response

  • 헤더필드 - 서버와의 통신에 필요한 여러 가지 정보를 표현

Request Header > Request-line

method / url(URI) / protocol ver

  • 3개의 필드로 구성
  • 각 필드의 구분자는 sp(공백)

Response Header > Response-line

protocol ver / status code / status string

  • 클라이언트의 요청에 대한 처리 결과
  • 위와 같이 3개의 필드로 구성함

개발자 도구 방법?

1. 크롬에서 오른쪽 위 점 3개 아이콘을 클릭 (F12)
2. Network 탭 이동
3. F5 또는 Ctrl+R 페이지 새로고침
4. name에 페이지 요청 목록 클릭
5. Header 탭 클릭 - 헤더 정보 확인
6. Response 정보도 확인

  • Get - 오픈된 상태
  • POST - 숨겨진 것

응답상태 코드?

  • 1XX : 요청 받았음, 작업 진행 중
  • 2XX : 서버가 요청을 수락했음
  • 3XX : 요청에 문제 없음, 요청에 대한 선택 처리
    ※ 301, 302는 리디렉션(Redirection) 관련
  • 4XX : 클라이언트 요청 오류
    ※ 400 : Bad Request : 오탈자, 없는 리소스
    ※ 401 Unauthorized, 403 Forbidden
    ※ 404 Not Found : 리소스를 찾을 수 없음
  • 5XX : 서버 에러

프론트 엔드 / 백엔드?

프론트 엔드 : 사용자가 볼 수 있는 화면, 즉 사용자 인터페이스(User Interface, UI)를 일컫음

백 엔드 : 웹 애플리케이션의 사용자가 보지 못하는 영역인 서버나 데이터베이스를 관리하는 기술
